medcode 医疗专病大模型02
medcode 是什么
代码范式的医疗专病大模型基座
  • 不是普通的自然语言医疗问答，而是把临床决策用"代码"表达
  • 决策路径无自然语言生成 —— 幻觉被结构性消除
三件套基座架构
代码大模型
基于代码范式预训练 / 微调，决策本身用代码表达，而非自然语言推理。
+
医疗 toolkit
可调用的结构化工具集：剂量计算、分期判定、禁忌检查、规则查询。
+
医疗 harness
执行 / 评估环境：规则 fire、abstain 逻辑、回归测试、临床真值对照。
三件套基于真实反馈共同迭代——模型不再独自承担决策能力，构成可演进的医疗 agent 基座。
medcode 医疗专病大模型03
核心技术
把指南"编译"成可执行规则
  • 权威指南（如 NCCN 癌种） → 注册式 Python 规则
  • 规则在验真上下文才 fire，否则 abstain（不硬答）
  • 封闭类型 ClinicalAction；LLM 退到结构化抽取（ETL），不参与决策
  • 指南合规烧进权重，不是 runtime RAG 兜底
一份规则，两个身份
运行时：临床安全护栏，保证决策合规、可审计。
训练时：代码化规则回训模型 —— 形成数据飞轮，指南能力沉淀进模型本身。
为什么能消除幻觉
决策路径没有自由文本生成，只有"规则命中 / 弃权"两种结果，每一步都可回溯到具体规则与证据。

方法论 · 度量衡
为什么 medcode 会越用越准
  • 先给"好不好"造一把游标卡尺——把主观评价变成可复算的量化标尺
  • 有了标尺，才能持续教模型并验收，每一版都知道进步在哪、退步在哪
① 校准
用 Rubric（维度 × 档位）取代拍脑袋打分，把临床好坏拆成可判定的标准。
→
② 教模型
SFT / 蒸馏 / On-Policy Distillation / DPO / RL 逐级把专病能力灌进权重。
→
③ 验收
GSB 对比 + Failure Code 归因，回归时精准定位"哪类 case 退步"。
三件套 + 度量衡构成数据飞轮：真实临床反馈 → 打标 → 回训 → 再评测，指南与专病能力持续沉淀进模型本身。
